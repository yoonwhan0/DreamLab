4. 신규 데이터 기본 규칙
신규 생성 데이터의 출처는 seed로 입력한다.

예: DreamLab synthetic seed v2.0

이메일은 실제 개인정보가 아닌 example.com 테스트 이메일을 사용한다.
예: seed001@example.com
기존 데이터와 신규 데이터의 문서ID가 중복되지 않도록 검사한다.

제목

꿈의 핵심 장면이 바로 이해되는 자연스러운 제목으로 작성한다.

너무 길거나 꿈내용 전체를 그대로 복사한 제목은 피한다.

꿈내용
최소 2문장 이상 작성한다.
등장인물, 장소, 행동, 변화, 감정을 포함한다.
장면의 시작과 전환, 결말이 어느 정도 드러나야 한다.
실제 꿈처럼 비현실적인 요소가 자연스럽게 포함될 수 있다.
감정

영문 감정 태그를 쉼표로 구분해 2개 이상 작성한다.

예시:

happy, calm
scared, relieved
sad, nostalgic
anxious, hopeful

꿈내용에서 실제로 느껴지는 감정과 일치해야 한다.

분류

영문 분류 태그를 |로 구분한다.

예시:

family|love|general
work|anxiety|general
animal|nature|fantasy
money|achievement|general

꿈내용과 관계없는 분류를 임의로 넣지 않는다.

키워드

꿈의 검색 및 분석에 사용할 수 있는 핵심어를 한글로 4~6개 작성하고 쉼표로 구분한다.

앵커

꿈 전체를 대표하는 가장 중요한 상징이나 장면 하나를 짧은 명사구로 작성한다.

클러스터

앵커가 의미하는 심리적·상황적 맥락을 자연스러운 한글 문구로 작성한다.

꿈속 핵심 장면을 발생 순서대로 2~4개 추출하고 |로 구분한다.

꿈내용에 없는 장면을 새롭게 만들어 넣지 않는다.

일반해몽
3문장 내외로 작성한다.
일반적인 상징 해석과 감정 흐름을 설명한다.
특정 사건이 반드시 발생한다고 단정하지 않는다.
복권, 사고, 죽음, 질병, 임신 등의 결과를 확정적으로 예언하지 않는다.
연구소입장
DreamLab만의 분석 관점으로 3~4문장 작성한다.
길몽이나 흉몽으로 단순하게 단정하지 않는다.
상징 자체보다 감정, 최근 생활, 반복 행동, 30일 후 변화 관찰을 강조한다.
동일한 시작 문장을 과도하게 반복하지 않는다.
상징

꿈속 주요 상징이 어떤 심리적·상황적 의미를 가질 수 있는지 설명한다.

검증되지 않은 사용자 통계나 연구 결과를 사실처럼 작성하지 않는다.
